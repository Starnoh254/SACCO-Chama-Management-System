# SACCO Management System

## Overview

The SACCO Management System is a backend REST API built using Java Spring Boot for managing SACCO operations digitally.

It provides functionality for managing:

- Members
- Contributions
- Loan applications
- Loan repayments
- Notifications
- Audit logs
- Financial reports

The system is designed to help SACCO administrators manage members, track financial transactions, monitor loans, and generate reports in a centralized and structured way.

---

# Why this project exists

This project exists to digitize core SACCO operations that are often difficult to manage manually.

Many SACCO processes involve:

- registering members
- tracking contributions
- issuing loans
- recording repayments
- generating reports
- maintaining financial accountability

Managing these processes manually can become slow, repetitive, and error-prone.

This system solves that problem by providing a backend API that stores and manages SACCO data consistently and securely.

The goal is to:

- reduce manual record keeping
- improve financial tracking
- improve accountability
- simplify member management
- support report generation
- provide a scalable backend foundation for SACCO operations

# How to run the project

## 1. Clone the project

```bash
git clone <your-repository-url>
cd sacco-management
```

---

## 2. Configure the database

Create a PostgreSQL database:

```sql
CREATE DATABASE sacco_management;
```

---

## 3. Configure application properties

Update:

```properties
src/main/resources/application.properties
```

Example:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/sacco_management
spring.datasource.username=postgres
spring.datasource.password=your_password

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
```

---

## 4. Run the application

Using Maven:

```bash
./mvnw spring-boot:run
```

Or:

```bash
mvn spring-boot:run
```

The application will start on:

```text
http://localhost:8080
```

# Core Database Entities

The project currently manages the following core entities:

## Users

System users such as:

- administrators
- staff
- authorized system users

---

## Members

Stores SACCO member information such as:

- membership number
- national ID
- address
- membership status
- date joined

---

## Contributions

Tracks member contributions including:

- contribution amount
- contribution date
- payment method
- transaction reference
- payment status

---

## Loan Applications

Handles:

- loan requests
- requested amount
- interest rate
- loan term
- approval workflow

---

## Loan Repayments

Tracks:

- repayment amount
- payment date
- penalties
- remaining balance

---

## Notifications

Used for:

- contribution reminders
- loan updates
- announcements

---

## Audit Logs

Tracks system actions for accountability and traceability.

Examples:

- who approved a loan
- who updated a member
- when a contribution was recorded

---

## Financial Reports

Stores generated report metadata including:

- report type
- reporting period
- generated by
- generated time
- exported file path

# API Response Format

The project uses a consistent API response wrapper:

```json
{
  "success": true,
  "message": "Request successful",
  "data": {}
}
```

Example error response:

```json
{
  "success": false,
  "message": "Member not found",
  "data": null
}
```

# Exception Handling

The project uses centralized exception handling with:

```java
@ControllerAdvice
```

via:

```text
GlobalExceptionHandler
```

Handled exceptions include:

- ResourceNotFoundException
- BadRequestException
- DuplicateResourceException
- UnauthorizedException
- ForbiddenException
- DataIntegrityViolationException
- Generic fallback Exception

This ensures clean and consistent API error responses across the application.
